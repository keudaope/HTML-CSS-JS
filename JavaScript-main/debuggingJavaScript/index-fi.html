By: Ashutosh Sharma

JOHDANTO

JavaScript maailman suosituin ohjelmointikieli ja sitä käytetään määrittämään web-sivujen logiikka sekä tekemään HTML-pohjaisia ohjelmia. Sitä käytetään myös palvelin puolen ohjelmointikirjasto NodeJS:ssä.

Koska kaikki suosituimmat selaimet tukevat JavaScriptiä, kaikissa niissä on mukana myös sisäänrakennetut kehitystyökalut. Firefoxissa on myös mukana suosittu verkkokehitys lisäosa.

PERUSVIRHEENKORJAUS

Virheet voidaan yleensä luokitella johonkin seuraavista:

  1. Syntaksi- tai tulkintavirheen

  2. Ajonaikaiset poikkeukset

  3. Väärä logiikka

Syntaksivirheet

Näitä ovat esimerkiksi yhteensopimattomat tai puuttuvat lainausmerkit, sulut ja aaltosulut, väärät kirjaimet, kirjoitusvirheet, laittomat merkit ja paljon muuta. Ne voidaan yleensä siepata helposti staattisilla koodianalysaattoreilla , kuten JSHint [2] ja JSLint [3]. Nämä työkalut analysoivat koodisi ennen sen suorittamista verkkosivulla ja osoittavat syntaksin ja muut yleiset virheet ennen kuin julkaiset koodisi verkkosivustollasi tai sovelluksessasi.

Ajonaikaiset poikkeukset

Nämä ovat virheitä, jotka tapahtuvat, kun JavaScript-koodisi suoritetaan. Tällaisia virheitä voidaan laukaista viittaamalla määrittelemättömään muuttujaan, jakamalla nolla, epäonnistuneella "assert"-lauseella tai käyttämällä "throw"-lausetta koodissasi (tai JavaScript-kirjaston) määrittämään manuaalisesti, että poikkeus on tapahtunut.

Kun ajonaikainen poikkeus tapahtuu, mitään kyseisen koodirivin jälkeistä JavaScript-koodia ei suoriteta. Jos ajonaikaisia poikkeuksia ei siis havaita ja käsitellä asianmukaisesti, ne voivat jättää verkkosivusi tai sovelluksesi odottamattomaan tilaan.

Voit ottaa kiinni ja käsitellä ajonaikaisia poikkeuksia käärimällä koodisi (joka saattaa aiheuttaa poikkeuksen) tekstillä "try {}" ja määrittämällä käsittelijän "catch(exception) {}":

try {
var v = dummy: // causes a runtime exception
} catch(e) {
console.log(e.message); // "dummy is not defined"
}

Useimmissa selaimissa voit myös määrittää yleisen käsittelijän kaikille poikkeuksille, joita ei ole saatu kiinni "try… catch" -lohkon avulla, määrittämällä window.onerror:

window.onerror = function(errorMessage, url, lineNumber) {
// ... }

Google Chromessa voit asettaa DevToolsin keskeyttämään JavaScriptin suorittamisen kaikissa ajonaikaisissa poikkeuksissa napsauttamalla Lähteet-paneelin alaosassa olevaa keskeytyspainiketta. (Voit käynnistää DevToolsin painamalla Cmd-Opt-I (Mac) tai F12 (Windows) tai Valikko > Työkalut > Kehittäjätyökalut.) Voit keskeyttää JavaScriptin suorittamisen vain havaitsemattomissa suoritusaikapoikkeuksissa napsauta "tauko"-painiketta uudelleen. Napsauttaminen uudelleen poistaa keskeytyksen käytöstä poikkeusten yhteydessä.

Väärä logiikka

Virheellinen logiikka koodissasi ei näytä virheitä (esim. JavaScript-virhekonsolissa), mutta se saa koodisi toimimaan ei-haluamallasi tavalla. Tällaisten virheiden virheenkorjaus vaatii jonkin verran harjoittelua selaimesi tarjoamien virheenkorjaustyökalujen avulla.

Logiikkavirheiden virheenkorjaus sisältää yleensä seuraavat asiat:

  1. Lokit ja väitteet

Useimmat selaimet antavat sinun lisätä lokilausekkeita koodiisi, jotta voit kerätä hyödyllistä tietoa JavaScriptin suorittamisesta:

console.log("Mouse coordinates: " + evt.pageX + ", " + evt.pageY);

Google Chromessa tämä tulos näkyy DevToolsin Console -paneelissa. "Lokin" lisäksi voit käyttää myös "varoita", "virhe" ja "debug". Nämä eri luokat voidaan suodattaa DevToolsin konsolissa.
